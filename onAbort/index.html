onAbort()

Environment: client.

The client-side onAbort() hook is called whenever telefunction calls fail because of throw Abort().

It does not run when other errors are thrown, see Docs > Error Handling.

onAbort gives you type-safe access to any data from throw Abort(errorData) on the client. It is designed to be used as part of Telefunc's DRY security pattern. For more, see Docs > Permissions.

Use the abortValue property to access the error data.
