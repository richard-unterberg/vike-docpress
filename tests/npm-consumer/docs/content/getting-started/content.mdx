# Getting Started

This fixture exists to verify that a consumer can install `@unterberg/nivel` as an external package and run without any workspace linking.

<Alert heading="What this tests">
  The app uses the packaged entry points from a standalone install, keeps the consumer shell files visible in <code>pages/</code>, runs <code>nivel prepare</code>, and builds a docs page from a minimal docs graph.
</Alert>

## Current setup

- The fixture points to the local engine package using <code>file:../../packages/engine</code>.
- The consumer lives under <code>tests/npm-consumer</code> and is not part of the workspace packages list.
- Tailwind is still configured in a hand-authored consumer stylesheet.
- The authored docs config imports helpers from <code>@unterberg/nivel/config</code>.
- The Vike setup keeps a normal-looking <code>+config.ts</code> and spreads in <code>@unterberg/nivel/vike</code>.
